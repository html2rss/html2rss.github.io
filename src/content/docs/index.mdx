`html2rss` turns any website into a stable RSS feed. Run the 10-minute quickstart to see it in action, then move to a production setup when you are ready to stay updated.

## Start Here

**Quickstart:** [Run html2rss-web with Docker](/web-application/getting-started/)

The canonical path to your first feed:

1.  **Phase 1: Local Demo** — Run the stack and generate a feed from any page URL in minutes.
2.  **Phase 2: Production** — Move to [Deployment](/web-application/deployment/) for a stable, production-ready instance.
3.  **Phase 3: Refinement** — Create [Custom Feeds](/creating-custom-feeds/) only when you need absolute control over extraction.

## What People Use It For

- Follow blogs and news sites without social media algorithms.
- Track product updates, changelogs, and release notes.
- Monitor job postings and community forums.
- Subscribe to any site that does not publish its own feed.

## Practical Notes

- **Start with Docker**: It is the recommended way to run `html2rss-web`.
- **Start with a Page URL**: Automatic generation is the primary workflow.
- **Use Included Configs**: Check the [Feed Directory](/feed-directory/) first; the site you want might already be covered.
